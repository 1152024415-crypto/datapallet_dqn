# 系统启动说明

## 项目概述
本系统包含多个服务器组件，用于处理传感器数据、图像采集和场景分类。

## 启动顺序

### 第一步：启动主服务器
```bash
python main.py
```

功能：

启动传感器数据服务器

启动WebSocket服务器（用于图片上传）

启动场景分类服务器

10秒后模拟触发拍照请求

### 第二步：启动SA模拟客户端（10秒内）
```bash
python server/client.py
```

说明：

模拟SA设备连接到WebSocket服务器

必须在主服务器启动后10秒内运行，这是暂时的，因为目前没有rule engine来触发拍照，所以在main.py里写了一个延迟拍照的函数，需要保证SA和服务器已建立连接。

用于建立连接以便接收拍照请求

### 第三步：启动传感器模拟客户端
```bash
python sensor_client.py
```

说明：

模拟Sensor Hub发送传感器数据

可在任意时间启动

持续发送模拟数据到传感器服务器
